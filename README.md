# 驾驶员表情识别系统

基于 CNN 卷积神经网络 + OpenCV 实现的人脸检测与表情识别项目，可完成静态图片识别、摄像头实时情绪检测，适用于智能驾驶、人机交互、情绪监测等场景。

## 项目简介

本项目使用 Keras + OpenCV 搭建轻量级自定义 CNN 模型，基于 FER2013 人脸表情数据集训练，能够检测人脸并识别 **7 种基础情绪**：生气、厌恶、恐惧、开心、伤心、惊讶、中性。

系统支持单张图片表情推理、摄像头实时检测识别，可用于疲劳驾驶监测、路怒情绪预警、课堂情绪分析等实际场景。

## 功能特点

- 人脸检测：基于 Haar 级分类器自动定位人脸区域
- 表情分类：识别 7 类基础人脸情绪
- 静态识别：对单张图片进行人脸表情预测并保存结果
- 实时检测：调用摄像头，实时标注人脸情绪
- 训练可视化：自动生成训练 Loss / 准确率曲线
- 数据集处理：支持 CSV 数据集转图片、自动分类存储

## 数据集说明

- 采用 **FER2013 人脸表情数据集**

- 样本总量：35887 张 48×48 灰度人脸图像

- 类别划分：7 种情绪标签

  

  0 - 生气  1 - 厌恶  2 - 恐惧  3 - 开心  4 - 伤心  5 - 惊讶  6 - 中性

- 划分方式：训练集 / 验证集按 9:1 划分

- 支持脚本将 CSV 原始数据转为可视图片

## 模型介绍

### 网络结构

自定义轻量级 SimpleCNN 卷积神经网络：

- 3 层卷积 + 平均池化组合
- 引入 PReLU 激活函数、BatchNormalization 批归一化
- 多层 Dropout 防止过拟合
- 两层全连接层，Softmax 输出 7 类概率

### 模型参数

- 输入尺寸：48×48×1 灰度图像
- 优化器：Adam
- 损失函数：Categorical Crossentropy
- 批次大小：128
- 训练轮数：1000 轮
- 总参数量：67231

### 训练效果

- 训练集准确率稳步提升，验证集最高约 40%
- 存在轻微波动，受数据集标注错误、类别不平衡影响
- 自动保存最优模型，支持断点续训

## 测试结果可视化

### a. 图像测试结果展示

使用 `image.py` 对指定图片进行测试，展示识别结果。

**功能**：从单张图片中检测人脸并识别表情。

**实现步骤**：

1. 使用 Haar Cascade 分类器检测人脸区域；
2. 将人脸图像缩放至 48×48 尺寸，输入模型进行预测；
3. 输出预测情绪标签，并在原图上绘制人脸框与情绪文字标注。

### b. 视频实时识别演示

基于 OpenCV 读取摄像头视频流，通过 `haarcascade_frontalface_default.xml` 完成人脸检测，裁剪人脸区域送入训练好的 CNN 模型，实现实时情绪预测。

- 运行 `video.py` 即可调用摄像头，实时检测人脸并标注对应情绪。
- **功能**：实时捕获摄像头画面，检测人脸并识别表情状态。

**实现步骤**：

1. 逐帧检测画面中的人脸，提取人脸区域特征；
2. 采用滑动窗口统计最近 10 帧出现频次最高的情绪；
3. 在视频画面上实时标注当前人脸情绪状态。

**演示效果**

- 实时帧率：CPU 环境下约 15fps；
- 识别表现：微笑识别为开心、皱眉多识别为生气或恐惧。

**视频画面为黑白灰度原因**

代码将摄像头每一帧转为灰度图，不使用彩图原因：

1. **减少计算量**

   灰度图为单通道，彩色图为三通道；灰度输入可降低模型计算维度与复杂度，提升推理速度，契合 CNN 模型 48×48×1 的标准输入格式。

2. **匹配训练数据格式**

   模型基于 FER2013 灰度数据集训练，仅支持单通道灰度输入；传入彩图会造成数据分布不一致，降低识别准确率。

3. **规避颜色干扰**

   表情识别依赖五官形态、眉眼嘴角变化，与色彩无关；灰度图可避免光照、肤色、环境色彩带来的噪声干扰，提升模型鲁棒性。

```
python image.py
```

```
python video.py
```

## 实验分析

1. 数据集存在标注错误、部分图像模糊，影响精度
2. 样本类别不均衡（厌恶样本最少）导致验证准确率波动
3. 48×48 小图像特征有限，复杂表情区分难度大
4. 可通过数据增强、更换 ResNet 等深层网络进一步提升准确率
